## 一、架构概览

### 1.1 项目定位

**系统类型**: Crypto Market Intelligence System（加密市场情报系统）+ 量化交易信号生成引擎

**核心能力**:
- 多源市场数据采集（价格、新闻、社交媒体、ETF、宏观）
- 事件驱动架构（Kafka-based Event Streaming）
- 智能信号融合（Multi-source Signal Fusion）
- LLM 增强分析（Sentiment Analysis & Event Extraction）

### 1.2 技术栈

| 层级 | 技术选型 | 评估 |
|------|---------|------|
| **语言** | Python 3.10+ | ✅ 现代异步支持 |
| **Web框架** | FastAPI | ✅ 高性能、自动 OpenAPI |
| **消息队列** | Kafka (FastStream) | ✅ 事件驱动核心 |
| **缓存** | Redis | ✅ 完整集成 |
| **时序存储** | ClickHouse | ✅ 适合时序数据 |
| **ORM** | SQLAlchemy | ✅ 完整支持 |
| **LLM** | OpenAI/Anthropic/MiniMax Pool | ✅ 多模型支持 |
| **监控** | 日志基础设施完整 | ⚠️ 缺少 Metrics/Tracing |
| **容器化** | Docker + Docker Compose | ✅ 完整支持 |

#### 3.3.2 融合流程 ([engine.py](file:///Users/yangdezeng/00_crypto/00_trade_agent/20260506/backend/services/fusion_service/engine.py))

```python
def process(self, price_change: float = 0.0) -> list[FusionSignal]:
    # 1. 获取有效事件
    valid_events = self.buffer.get_valid()
    
    # 2. 事件聚合
    groups = self.aggregator.aggregate(valid_events)
    
    # 3. 评分
    for group in groups:
        signal = self._generate_signal(group, price_change)
    
    # 4. 冷却机制（避免重复）
    if self.last_signal_time:
        elapsed = (now - self.last_signal_time).total_seconds()
        if elapsed < self.signal_cooldown_seconds:
            return []
```

**评估**:
- ✅ 时间窗口聚合
- ✅ 置信度评分
- ✅ 冷却机制
- ⚠️ 简单加权平均，未用 ML
- ⚠️ 未利用价格信息强化信号
