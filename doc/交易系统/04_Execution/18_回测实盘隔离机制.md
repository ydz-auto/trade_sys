# 回测/实盘隔离机制设计文档

# 🧠 1. 核心概念

# 1.1 为什么必须隔离

```
❌ 如果回测和实盘共用同一套状态/配置/执行
→ 回测结果不可信
→ 实盘可能爆仓
→ 策略参数泄露
```

# 1.2 隔离目标

```
✅ 数据隔离 → 回测用历史数据，实盘用实时数据
✅ 状态隔离 → 回测状态不影响实盘状态
✅ 配置隔离 → 回测参数和实盘参数独立
✅ 执行隔离 → 回测不真实下单，实盘真实下单
✅ 存储隔离 → 回测日志和实盘日志分开
```

class BacktestStateManager:
    def reset_state(self):
        self.state = {
            "balance": self.initial_balance,
            "positions": [],
            "orders": [],
            "trades": [],
            "pnl_history": []
        }

    def take_snapshot(self) -> dict:
        return {
            "balance": self.state["balance"],
            "positions": copy.deepcopy(self.state["positions"]),
            "timestamp": time.time()
        }

# 🚀 5. 执行隔离

# 5.1 执行模式判断

```python
class ExecutionModeGuard:
    def __init__(self, mode: str):
        self.mode = mode

    def can_execute_real_order(self) -> bool:
        return self.mode == "LIVE"

    def should_send_to_exchange(self) -> bool:
        return self.mode == "LIVE"

    def should_simulate_fill(self) -> bool:
        return self.mode in ["BACKTEST", "SIMULATION"]
```

# 🚨 10. 常见错误

```
❌ 共用同一Redis实例存储状态
❌ 回测和实盘使用相同API Key
❌ 回测时修改了实盘配置
❌ 回测订单混入了实盘订单表
❌ 模式切换时未检查仓位
```
