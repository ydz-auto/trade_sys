# 🧠 一、模块定位

## 1.1 核心问题

```
不是所有信号都一样好

BUY → 立即执行 ❌
BUY → 质量评分0.9 → 执行 ✅
BUY → 质量评分0.3 → 等待/观察 ✅
```

## 1.2 作用

信号质量评分是**交易决策的过滤器**和**执行优先级的依据**：
- 过滤低质量信号
- 决定仓位大小
- 决定执行延迟容忍度

## 1.3 在系统中的位置

```
Factor → Risk → Decision → Signal Quality → Position → Execution
                              ↑
                        Regime Engine
```

# 🧮 四、详细计算逻辑

## 4.1 因子一致性（Factor Alignment）

### 4.1.1 概念

有多少个因子指向同一个方向？

```
trend = +0.6 (bullish)
flow = +0.3 (slightly bullish)
sentiment = -0.2 (bearish)
macro = +0.4 (bullish)
behavioral = +0.5 (bullish)

→ 4/5 因子指向 bullish
→ alignment_score = 0.8
```

### 4.1.2 计算

```python
def compute_factor_alignment(factor_scores, signal_direction):
    """
    factor_scores: {
        "trend": 0.6,
        "flow": 0.3,
        "sentiment": -0.2,
        "macro": 0.4,
        "behavioral": 0.5
    }
    signal_direction: "LONG" or "SHORT"
    """
    aligned_count = 0
    total_count = 0

    for factor, score in factor_scores.items():
        total_count += 1

        if signal_direction == "LONG" and score > 0.1:
            aligned_count += 1
        elif signal_direction == "SHORT" and score < -0.1:
            aligned_count += 1
        elif abs(score) < 0.1:
            aligned_count += 0.5  # 中性因子不扣分

    alignment_score = aligned_count / total_count

    # 额外加权：强因子权重更高
    weighted_alignment = 0
    total_weight = 0
    for factor, score in factor_scores.items():
        weight = FACTOR_WEIGHTS.get(factor, 1.0)
        direction_score = score if signal_direction == "LONG" else -score
        weighted_alignment += max(0, direction_score) * weight
        total_weight += weight

    return (alignment_score + weighted_alignment) / 2
```

## 4.2 信号强度（Signal Strength）

### 4.2.1 概念

信号的绝对强度，不是刚好过零，而是有明确方向。

```
composite_score = 0.15 → 刚好过零，很弱
composite_score = 0.65 → 明确方向，很强
```

### 4.2.2 计算

```python
def compute_signal_strength(composite_score):
    """
    composite_score: -1 to 1
    """
    abs_score = abs(composite_score)

    # 非线性映射：接近0的分数要大幅打折
    if abs_score < 0.3:
        strength = abs_score * 0.5  # 弱信号大幅削弱
    elif abs_score < 0.5:
        strength = 0.15 + (abs_score - 0.3) * 0.5
    elif abs_score < 0.7:
        strength = 0.25 + (abs_score - 0.5) * 1.0
    else:
        strength = 0.45 + (abs_score - 0.7) * 1.5
        strength = min(strength, 1.0)

    return strength
```

## 4.3 Regime 匹配度（Regime Match）

### 4.3.1 概念

信号方向与当前市场状态是否匹配。

```
信号 LONG + 市场 PANIC → 极度不匹配
信号 LONG + 市场 TRENDING → 完美匹配
```

### 4.3.2 匹配矩阵

```python
REGIME_MATCH_MATRIX = {
    ("LONG", "TRENDING"): 1.0,
    ("LONG", "RANGE"): 0.5,
    ("LONG", "PANIC"): 0.1,
    ("LONG", "EUPHORIA"): 0.4,
    ("LONG", "RISK_OFF"): 0.2,
    ("LONG", "UNCERTAIN"): 0.3,

    ("SHORT", "TRENDING"): 0.4,
    ("SHORT", "RANGE"): 0.5,
    ("SHORT", "PANIC"): 0.5,
    ("SHORT", "EUPHORIA"): 1.0,
    ("SHORT", "RISK_OFF"): 0.3,
    ("SHORT", "UNCERTAIN"): 0.3,

    ("HOLD", "ANY"): 1.0,
}
```

### 4.3.3 计算

```python
def compute_regime_match(signal_direction, current_regime, confidence):
    """
    confidence: 当前regime的置信度
    """
    base_match = REGIME_MATCH_MATRIX.get((signal_direction, current_regime), 0.5)

    # 如果regime置信度高，匹配度更重要
    if confidence > 0.8:
        regime_match = base_match
    elif confidence > 0.6:
        regime_match = 0.5 + base_match * 0.5
    else:
        regime_match = 0.7 + base_match * 0.3  # 置信度低时，不确定性折扣

    return regime_match
```

## 4.4 风险调整（Risk Adjusted）

### 4.4.1 概念

考虑当前风险水平的信号质量。

### 4.4.2 计算

```python
def compute_risk_adjusted(quality_score, risk_index, risk_drivers):
    """
    quality_score: 原始信号质量分
    risk_index: 0-100
    risk_drivers: 当前主要风险因素
    """
    # 基础风险折扣
    risk_factor = 1 - (risk_index / 100) * 0.5  # 最高50%折扣

    # 额外折扣：特定风险因素
    risk_discount = 1.0

    if "volatility_spike" in risk_drivers:
        risk_discount *= 0.8

    if "ETF_outflow" in risk_drivers and signal_direction == "LONG":
        risk_discount *= 0.85

    if "negative_news" in risk_drivers:
        risk_discount *= 0.85

    if "gold_surge" in risk_drivers and asset in ["BTC", "ETH"]:
        risk_discount *= 0.80

    # 组合
    final_risk_factor = risk_factor * risk_discount

    return quality_score * max(0.3, final_risk_factor)  # 最低30%
```

## 4.5 时机（Timing）

### 4.5.1 概念

信号产生后多久执行？市场是否还在信号有效窗口内？

### 4.5.2 因素

```python
TIMING_FACTORS = {
    # 1. 信号延迟（从产生到现在的时长）
    "signal_age": {
        "0-5min": 1.0,
        "5-15min": 0.9,
        "15-30min": 0.7,
        "30-60min": 0.5,
        ">60min": 0.3,
    },

    # 2. 时间框架一致性
    "timeframe_consistency": {
        "1h": 1.0,
        "4h": 0.9,
        "1d": 0.8,
        "1m": 0.5,  # 短期噪音
        "1w": 0.7,
    },

    # 3. 事件风险窗口
    "event_window": {
        "normal": 1.0,
        "near_high_impact": 0.6,
        "during_high_impact": 0.3,
    }
}
```

### 4.5.3 计算

```python
def compute_timing(signal_age_minutes, timeframe, upcoming_events):
    # 年龄折扣
    age_score = 1.0
    for threshold, score in TIMING_FACTORS["signal_age"].items():
        if "min" in threshold:
            mins = int(threshold.replace("min", ""))
            if signal_age_minutes <= mins:
                age_score = score
                break
        elif ">" in threshold:
            mins = int(threshold.replace(">min", ""))
            if signal_age_minutes > mins:
                age_score = score

    # 时间框架折扣
    tf_score = TIMING_FACTORS["timeframe_consistency"].get(timeframe, 0.7)

    # 事件窗口折扣
    event_score = 1.0
    if upcoming_events:
        if any(e.severity > 0.8 for e in upcoming_events):
            event_score = 0.3
        elif any(e.severity > 0.5 for e in upcoming_events):
            event_score = 0.6

    return (age_score * 0.4 + tf_score * 0.4 + event_score * 0.2)
```

## 4.6 基本面支撑（Fundamental Support）

### 4.6.1 概念

是否有基本面因素支撑这个信号？短期技术信号是否有长期逻辑？

### 4.6.2 评分因素

```python
FUNDAMENTAL_FACTORS = {
    # 1. 趋势持续性
    "trend_persistence": {
        "strengthening": 1.0,
        "stable": 0.8,
        "weakening": 0.4,
    },

    # 2. 资金流方向
    "flow_direction": {
        "inflow_sustained": 1.0,
        "inflow_new": 0.8,
        "outflow": 0.3,
    },

    # 3. 宏观环境
    "macro_environment": {
        "supportive": 1.0,
        "neutral": 0.7,
        "adverse": 0.3,
    },

    # 4. ETF情绪
    "etf_sentiment": {
        "strong_inflow": 1.0,
        "mild_inflow": 0.7,
        "neutral": 0.5,
        "outflow": 0.2,
    }
}
```

### 4.6.3 计算

```python
def compute_fundamental_support(fundamental_data):
    scores = []

    # 趋势持续性
    trend_trend = fundamental_data.get("trend_trend", "stable")
    scores.append(FUNDAMENTAL_FACTORS["trend_persistence"].get(trend_trend, 0.5))

    # 资金流
    flow = fundamental_data.get("etf_flow", "neutral")
    scores.append(FUNDAMENTAL_FACTORS["flow_direction"].get(flow, 0.5))

    # 宏观
    macro = fundamental_data.get("macro_env", "neutral")
    scores.append(FUNDAMENTAL_FACTORS["macro_environment"].get(macro, 0.5))

    # ETF情绪
    etf = fundamental_data.get("etf_sentiment", "neutral")
    scores.append(FUNDAMENTAL_FACTORS["etf_sentiment"].get(etf, 0.5))

    return sum(scores) / len(scores)
```

# ⚖️ 五、综合评分计算

## 5.1 最终评分

```python
def compute_signal_quality(
    factor_scores,
    composite_score,
    signal_direction,
    current_regime,
    regime_confidence,
    risk_index,
    risk_drivers,
    signal_age_minutes,
    timeframe,
    upcoming_events,
    fundamental_data
):
    # 各维度评分
    f_alignment = compute_factor_alignment(factor_scores, signal_direction)
    f_strength = compute_signal_strength(composite_score)
    f_regime = compute_regime_match(signal_direction, current_regime, regime_confidence)

    # 风险调整（基于因子一致性）
    base_quality = (
        f_alignment * QUALITY_WEIGHTS["factor_alignment"] +
        f_strength * QUALITY_WEIGHTS["signal_strength"] +
        f_regime * QUALITY_WEIGHTS["regime_match"]
    )

    f_risk = compute_risk_adjusted(base_quality, risk_index, risk_drivers)
    f_timing = compute_timing(signal_age_minutes, timeframe, upcoming_events)
    f_fundamental = compute_fundamental_support(fundamental_data)

    # 综合评分
    overall = (
        base_quality * 0.60 +  # 基础质量
        f_risk * QUALITY_WEIGHTS["risk_adjusted"] +
        f_timing * QUALITY_WEIGHTS["timing"] +
        f_fundamental * QUALITY_WEIGHTS["fundamental_support"]
    )

    return {
        "overall_score": round(overall, 2),
        "components": {
            "factor_alignment": round(f_alignment, 2),
            "signal_strength": round(f_strength, 2),
            "regime_match": round(f_regime, 2),
            "risk_adjusted": round(f_risk, 2),
            "timing": round(f_timing, 2),
            "fundamental_support": round(f_fundamental, 2),
        }
    }
```

## 6.2 信号去抖（防止抖动）

```python
class SignalSmoother:
    def __init__(self, confirmations_needed=2):
        self.confirmations_needed = confirmations_needed
        self.signal_history = []

    def should_execute(self, signal, quality_score):
        """
        只有连续N次高质量信号才执行
        防止：BUY → SELL → BUY 的震荡
        """
        self.signal_history.append({
            "direction": signal.direction,
            "quality": quality_score,
            "timestamp": time.time()
        })

        # 只保留最近N次
        self.signal_history = self.signal_history[-self.confirmations_needed:]

        if len(self.signal_history) < self.confirmations_needed:
            return False, "need_more_confirmations"

        # 检查是否所有信号方向一致
        directions = [s["direction"] for s in self.signal_history]
        if len(set(directions)) > 1:
            return False, "signal_reversal"

        # 检查是否所有信号都高质量
        if all(s["quality"] >= 0.65 for s in self.signal_history):
            return True, "confirmed"
        else:
            return False, "quality_too_low"
```

    recent = [s for s in signal_history

# 🔧 八、动态权重调整

## 8.1 基于历史数据优化

```python
def optimize_quality_weights(signal_history):
    """
    定期优化各维度权重
    找出哪个维度最能预测盈利
    """
    executed = [s for s in signal_history if s["executed"] and s["outcome"]]

    if len(executed) < 100:
        return QUALITY_WEIGHTS  # 数据不足不调整

    results = {}
    for dimension in ["factor_alignment", "signal_strength", "regime_match",
                       "risk_adjusted", "timing", "fundamental_support"]:
        scores = [s["components"][dimension] for s in executed]
        pnls = [s["outcome"]["pnl"] for s in executed]
        correlation = np.corrcoef(scores, pnls)[0][1]
        results[dimension] = correlation

    # 调整权重：相关性高的维度权重增加
    total_corr = sum(max(0, c) for c in results.values())
    if total_corr > 0:
        adjusted = {}
        for dim, corr in results.items():
            if corr > 0:
                adjusted[dim] = max(0.05, QUALITY_WEIGHTS[dim] * (1 + corr))
            else:
                adjusted[dim] = QUALITY_WEIGHTS[dim]

        # 归一化
        total = sum(adjusted.values())
        return {k: v / total for k, v in adjusted.items()}

    return QUALITY_WEIGHTS
```

# ⚠️ 九、常见错误

| 错误 | 后果 | 正确做法 |
|------|------|----------|
| 不做信号质量过滤 | 执行很多低质量信号 | 必须有质量门槛 |
| 权重固定不变 | 适应不了市场变化 | 动态调整权重 |
| 不考虑Regime匹配 | 逆势交易 | Regime权重最高 |
| 不做信号去抖 | 频繁止损 | 连续确认机制 |
| 只看强度不看质量 | 刚好过零的信号也执行 | 非线性强度映射 |
