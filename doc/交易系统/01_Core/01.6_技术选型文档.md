# TradeAgent 技术选型文档

---

# 🧠 一、技术选型概览

```
TradeAgent = React前端 + Python后端 + 多数据库 + Docker部署
```

---

# 🖥️ 二、前端技术栈（React）

| 模块/功能 | 技术选型 | 免费/收费 | 说明 |
|-----------|----------|-----------|------|
| 核心框架 | **React 18+** | ✅ 免费开源 | 组件化开发 |
| 状态管理 | **Redux Toolkit** | ✅ 免费开源 | 全局状态，支持 WebSocket 数据流 |
| UI 组件库 | **Ant Design** | ✅ 免费开源 | 表格、表单、布局 |
| 样式 | **Tailwind CSS**（可选） | ✅ 免费开源 | 快速布局、样式统一 |
| 数据可视化 | **ECharts / Recharts** | ✅ 免费开源 | 图表展示行情、因子、回测曲线 |
| 实时数据 | **WebSocket** | ✅ 免费 | 推送行情/ETF/新闻/信号 |
| 路由 | **React Router** | ✅ 免费开源 | 页面导航 |
| 构建工具 | **Vite / Webpack** | ✅ 免费开源 | 打包和热更新 |

---

# ⚙️ 三、后端技术栈

| 模块/功能 | 技术选型 | 免费/收费 | 说明 |
|-----------|----------|-----------|------|
| 核心语言 | **Python 3.10+** | ✅ 免费开源 | 策略开发、数据处理 |
| 高性能模块 | **C++（可选）** | ✅ 免费 | 高频计算或撮合引擎加速 |
| 数据处理 | **Pandas / NumPy** | ✅ 免费开源 | 特征、因子计算 |
| 异步任务 | **Celery / Asyncio** | ✅ 免费开源 | 异步任务、定时计算 |
| API 框架 | **FastAPI / Flask** | ✅ 免费开源 | 提供前端接口 |
| 消息队列 | **Kafka / RabbitMQ** | ✅ 开源免费 | 数据流、事件推送 |
| 交易接口 | **CCXT / 官方交易所 API** | ✅ 免费 | 统一交易所接口，支持现货/杠杆 |
| 回测框架 | **backtrader / zipline** | ✅ 免费开源 | 历史数据回测 |

---

# 💾 四、数据库与存储

| 类型 | 技术选型 | 免费/收费 | 说明 |
|------|----------|-----------|------|
| 关系型数据库 | PostgreSQL / PostgreSQL | ✅ 本地免费 | 账户、配置、策略元数据 |
| 列式数据库 | ClickHouse | ✅ 开源 | 高频行情、因子、交易数据分析 |
| 缓存 | Redis | ✅ 免费开源 | 实时仓位、信号、行情快速访问 |
| 时序数据库 | InfluxDB | ✅ 社区版免费 | 历史行情、指标存储 |
| 对象存储 | MinIO / 本地存储 | ✅ 免费 | 回测数据、日志文件 |
| 文件/备份 | 本地 / MinIO | ✅ 免费 | 数据备份与存档 |

> 云托管服务（RDS、ClickHouse Cloud、Redis Cloud、S3/OSS）可能收费，本地完全可免费。

| 模块 | 技术选型 | 免费/收费 | 说明 |
|------|----------|-----------|------|

| 证书/安全 | Let's Encrypt | ✅ 免费 | HTTPS、API 安全 |

# 🔗 七、模块与数据库对应

| 模块 | 主要存储 | 说明 |
|------|----------|------|
| 用户/账户 | PostgreSQL | 关系型数据 |
| 行情数据 | ClickHouse | 高频写入、分析查询 |
| 实时状态 | Redis | 缓存、实时数据 |
| 历史K线 | InfluxDB | 时序数据 |
| 因子计算 | ClickHouse + Redis | 计算结果存储 |
| 回测数据 | ClickHouse + MinIO | 大规模历史数据 |
| 日志 | ELK Stack | 日志收集与分析 |
| 配置文件 | PostgreSQL | 持久化配置 |
| 策略状态 | Redis + PostgreSQL | 运行时状态 + 持久化 |

# 📋 九、技术选型决策

| 场景 | 推荐技术 | 理由 |
|------|----------|------|
| 快速验证 | PostgreSQL + Redis + Docker | 简单易部署 |
| 性能优先 | ClickHouse + Kafka | 高频数据处理 |
| 生产环境 | K8s + 完整监控 | 可扩展、高可用 |
| 成本敏感 | 全本地部署 | 完全免费 |
