# 简化架构说明

## 核心九层

| 层级 | 核心功能 | 关键技术 |
|------|----------|----------|
| **数据层** | 交易所行情、ETF资金流、新闻事件、宏观数据 | TDP标准化 |
| **事件层** | 语义理解、数据 → 事件转换 | LLM / 规则解析 |
| **融合层** | 多源信息融合、置信度计算、信息验证 | 投票/加权/贝叶斯 |
| **特征层** | 收益率、波动率、动量、情绪、宏观指标 | 特征计算 |
| **因子层** | Trend、Flow、Sentiment、Macro因子 | 多因子模型 |
| **风险引擎** | 风险指数、杠杆控制、动态止损止盈 | 实时风控 |
| **决策引擎** | 多因子决策、多资产联动 | 规则+量化 |
| **仓位引擎** | 仓位计算、动态调整 | 杠杆管理 |
| **执行模块** | 模拟下单、回测、实盘隔离 | 执行引擎 |

# 数据流详解

```
1. 数据采集
   交易所API / ETF网站 / 新闻RSS / 宏观数据
          ↓
2. 数据标准化（TDP协议）
   统一格式、统一时间戳
          ↓
3. 事件化（Event）
   新闻 → ETF_INFLOW / NEWS_SENTIMENT 等结构化事件
          ↓
4. 融合（Fusion）
   多源事件 → 置信度 → 融合信号
          ↓
5. 特征计算
   价格特征 / 资金流特征 / 情绪特征
          ↓
6. 因子合成
   Trend + Flow + Sentiment + Macro
          ↓
7. 风险计算
   risk_index → max_leverage → SL/TP
          ↓
8. 决策生成
   信号：BUY / SELL / HOLD
          ↓
9. 仓位计算
   size × leverage → 最终下单
          ↓
10. 执行
   回测：历史数据模拟
   实盘：API下单
```

# 一句话总结

> 简化架构 = 从数据到执行的9个核心步骤，适合快速验证和理解系统核心逻辑。
> 新增的事件层 + 融合层是系统的"语义理解"核心，把原始数据转化为可计算的事件。
