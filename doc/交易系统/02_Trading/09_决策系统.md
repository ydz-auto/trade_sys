# 决策系统

<br />

***

# 🧠 一、核心结论（非常重要）

> ✅ **需要 LLM，但只能作为“辅助决策层”**
>
> ❗**最终交易信号必须由规则系统决定**

***

👉 标准架构：

```
Rule Engine（主决策）
    +
LLM（辅助判断 / 解释 / 微调）
```

***

# ⚠️ 二、为什么不能让 LLM 主导决策？

***

## ❌ 1. 不可控

```
同样输入 → 不同输出
```

***

## ❌ 2. 无法回测

```
历史数据 → 跑不出稳定结果
```

***

## ❌ 3. 无法精确执行

```
仓位 0.18？止损 2%？
```

👉 LLM做不到严格一致

***

# 🧩 三、正确架构（实盘级）

***

```
Factor Engine → 得分
        ↓
Risk Engine → 限制
        ↓
Rule Decision（主）
        ↓
LLM（辅助修正 / 解释）
        ↓
Position Engine → 执行参数
```

***

***

# 📄 四、决策系统设计（Decision Engine Spec）

***

# 4.1 输入

***

```json
{
  "composite_score": 0.35,
  "factors": {...},
  "risk_index": 65,
  "regime": "RISK_OFF"
}
```

***

***

# 4.2 核心输出

***

```json
{
  "signal": "BUY | SELL | HOLD",
  "confidence": 0.75
}
```

***

***

# 🧠 五、Rule Engine（主决策逻辑）

***

## 🎯 核心思想：

```
用“分数 + 风险”决定方向
```

***

## 5.1 基础规则

***

```
if composite_score > 0.3 and risk < 70:
    BUY

if composite_score < -0.3 and risk < 70:
    SELL

else:
    HOLD
```

***

***

## 5.2 风险过滤（必须）

***

```
if risk_index > 80:
    signal = HOLD
```

***

***

## 5.3 Regime 调整（重要）

***

```
RISK_OFF → 偏空
RISK_ON → 偏多
```

***

👉 示例：

```
if regime == RISK_OFF:
    提高 SELL 阈值
```

***

***

# 🧠 六、Confidence 计算（关键）

***

## 🎯 公式：

```
confidence = |composite_score| × (1 - risk_index/100)
```

***

👉 示例：

```
score = 0.5
risk = 50

confidence = 0.25
```

***

***

# 🤖 七、LLM 在决策系统中的正确用法

***

# ✅ 7.1 决策解释（强烈推荐）

***

LLM输入：

```
因子 + 风险 + 信号
```

***

输出：

```json
{
  "reason": "ETF持续流出 + 黄金上涨 → 风险厌恶",
  "confidence_adjustment": -0.05
}
```

***

***

# ✅ 7.2 微调信号（可选）

***

👉 LLM可以：

```
轻微调整 confidence
```

***

❗但不能：

```
改变 BUY → SELL
```

***

***

# ✅ 7.3 特殊事件覆盖（高级）

***

例如：

```
黑天鹅事件
```

***

LLM可以输出：

```json
{
  "override": "FORCE_HOLD"
}
```

***

👉 但必须受规则限制：

```
只允许收紧，不允许放宽
```

***

***

# ❌ 八、绝对禁止的设计

***

## ❌ LLM直接输出交易信号

```
“帮我判断是否买BTC”
```

***

## ❌ Agent自动交易

```
LLM → 交易所
```

***

## ❌ LLM决定仓位

```
仓位 = LLM输出
```

***

***
