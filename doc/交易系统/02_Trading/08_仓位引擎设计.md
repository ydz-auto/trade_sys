# 仓位引擎设计

<br />

# **📄 仓位引擎设计（Position Engine Spec）**

# **🧠 1. 模块定位**

# **1.1 作用**

将“交易信号”转化为可执行交易指令

# **1.2 输入 → 输出**

输入：

- 信号（方向 / 置信度）
- 风险指数
- 当前仓位
- 市场波动

输出：

- 开/平仓决策
- 仓位大小
- 杠杆倍数
- 止损 / 止盈

# **⚙️ 2. 输入定义（必须标准化）**

# **2.1 信号输入**

{

"signal": "LONG | SHORT | HOLD",

"confidence": 0.0 - 1.0

}

# **2.2 风险输入**

{

"risk\_index": 0 - 100

}

# **2.3 当前仓位**

{

"balance": 10000,

"positions": \[

{

"symbol": "BTC",

"direction": "LONG",

"size": 0.3,

"entry\_price": 68000,

"leverage": 3

}

]

}

# **2.4 市场波动（建议用ATR）**

{

"atr": 0.025   // 2.5%

}

# **🧮 3. 核心计算逻辑（最关键）**

# **3.1 基础仓位（Base Position Size）**

# **🎯 公式：**

base\_size = confidence × max\_position

# **示例：**

confidence = 0.8

max\_position = 0.3

→ base\_size = 0.24（24%仓位）

# **3.2 风险调整（Risk Adjustment）**

# **🎯 规则：**

risk\_factor = 1 - (risk\_index / 100)

# **示例：**

risk\_index = 70

risk\_factor = 0.3

# **调整后仓位：**

adjusted\_size = base\_size × risk\_factor

# **3.3 波动率调整（Volatility Adjustment）**

# **🎯 逻辑：**

波动越大 → 仓位越小

# **简化公式：**

vol\_factor = target\_vol / atr

（建议 target\_vol = 2%）

# **最终仓位：**

final\_size = adjusted\_size × vol\_factor

# **3.4 杠杆计算（Leverage）**

# **🎯 原则：**

高风险 → 低杠杆

高置信度 → 可适度提高

# **示例规则：**

if risk\_index > 70:

leverage = 1

elif confidence > 0.8:

leverage = 3

else:

leverage = 2

👉 建议初期限制：

max\_leverage = 3

# **🛑 3.5 止损（Stop Loss）**

# **🎯 使用ATR：**

stop\_loss = entry\_price × (1 - k × atr)

# **建议：**

k = 1.5 \~ 2

# **🎯 3.6 止盈（Take Profit）**

# **🎯 RR比：**

take\_profit = stop\_loss × RR

# **建议：**

RR = 2 \~ 3

# **🔁 4. 仓位冲突处理（非常关键）**

# **4.1 同方向**

已有多单 + 新信号多

→ 加仓（但不超过max\_position）

# **4.2 反方向**

已有多单 + 新信号空

→ 平仓 → 再开空

# **🚨 5. 风控规则（必须硬编码）**

# **❗ 规则1：最大仓位**

单资产 ≤ 30%

# **❗ 规则2：最大总风险**

总仓位 ≤ 50%

# **❗ 规则3：高风险限制**

risk\_index > 80 → 禁止开仓

# **❗ 规则4：必须止损**

无SL → 不允许下单

# **🧠 6. 输出格式（最终执行指令）**

{

"symbol": "BTC",

"action": "OPEN\_LONG",

"size": 0.18,

"leverage": 2,

"stop\_loss": 66000,

"take\_profit": 72000,

"confidence": 0.78,

"risk\_level": "medium"

}

# **🔄 7. 执行流程（完整）**

1. 获取信号
2. 获取风险指数
3. 获取当前仓位
4. 计算 base\_size
5. 风险调整
6. 波动调整
7. 计算杠杆
8. 设置止损止盈
9. 检查风控规则
10. 输出交易指令

# **⚠️ 8. 常见错误（必须避免）**

# **❌ 不设止损**

→ 必爆

# **❌ 仓位不随风险变化**

→ 回撤巨大

# **❌ 杠杆固定**

→ 无法适应市场

# **❌ 忽略已有仓位**

→ 重复开仓
