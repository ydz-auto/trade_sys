# 因子系统

好，这一版我帮你重写因子系统文档，做成更接近可实现 / 可回测 / 可扩展的工程级版本。

会把你刚刚问到的 LLM / 向量 / 多资产 全部正确嵌进去（但不会滥用）。

# **📄 因子系统设计（Factor System Spec v2）**

# **🧠 1. 系统定位**

# **1.1 定义**

因子系统是将多源市场数据 → 转换为标准化信号的核心模块

# **1.2 核心职责**

1. 数据 → 特征（Feature）
2. 特征 → 因子（Factor）
3. 因子 → 统一评分（Score）

# **1.3 输出（唯一标准接口）**

{

"factor\_scores": {

"trend": 0.55,

"flow": -0.2,

"sentiment": -0.6,

"macro": 0.3,

"behavioral": 0.4,

"historical": -0.1

},

"composite\_score": 0.18,

"regime": "RISK\_OFF"

}

# **🧩 2. 因子分类（严格分层）**

# **2.1 因子类型**

A. Quant Factors（量化因子）

- trend
- flow
- macro
- behavioral

B. Semantic Factors（语义因子）

- sentiment

C. Context Factors（上下文因子）

- historical（向量相似度）

# **⚠️ 设计原则**

Quant → 必须确定性计算

Semantic → 可用 LLM

Context → 只做辅助

# **📊 3. 数据输入标准（Feature Layer）**

# **3.1 时间统一**

所有因子基于统一时间窗口：

1m / 5m / 1h / 1d（可配置）

# **3.2 标准特征结构**

{

"price": {...},

"etf\_flow": {...},

"news": \[...],

"macro\_assets": {

"gold": {...},

"oil": {...}

},

"trader\_data": {...}

}

# **📈 4. Quant Factors（量化因子）**

# **4.1 Trend（趋势因子）**

# **输入**

- BTC / ETH 价格序列

# **特征**

returns（收益率）

MA（均线）

momentum（动量）

# **计算**

trend =

w1 \* momentum +

w2 \* MA\_signal +

w3 \* multi\_timeframe\_alignment

# **输出**

范围：\[-1, +1]

# **4.2 Flow（资金因子）**

# **输入**

- ETF资金流（每日）

# **特征**

netflow = inflow - outflow

flow\_change（变化率）

# **计算**

flow = tanh(netflow / scale) + α \* acceleration

# **4.3 Macro（宏观因子）**

# **输入**

- 黄金价格
- 原油价格

# **特征**

gold\_return

oil\_return

# **计算**

macro =

- w1 \* gold\_return
- w2 \* oil\_return

（负号：风险资产反向）

# **4.4 Behavioral（交易员因子）**

# **输入**

- 多空比
- 顶级交易员仓位

# **计算逻辑**

if retail\_long\_ratio 高 → bearish

if top\_trader\_long → bullish

# **输出**

behavioral = 加权结果

# **📰 5. Semantic Factor（语义因子）**

# **5.1 Sentiment（情绪因子）**

# **输入**

- 新闻（RSS / 抓取）

# **⚙️ 处理流程（唯一使用LLM的地方）**

新闻文本

↓

LLM解析

↓

结构化输出

# **输出格式**

{

"sentiment": -0.7,

"event\_type": "geopolitical",

"impact": 0.8,

"timestamp": 1710000000

}

# **时间衰减（必须）**

weight = exp(-λ \* 时间差)

# **最终计算**

sentiment = Σ(情绪 × 权重)

# **🧠 6. Context Factor（上下文因子）**

# **6.1 Historical（历史相似度）**

# **输入**

- 当前市场特征向量

# **流程（向量 + LLM）**

当前特征

↓

向量数据库

↓

检索历史相似片段

↓

LLM总结（可选）

# **输出**

{

"historical\_score": -0.2,

"scenario": "risk\_off"

}

# **⚠️ 限制**

权重 ≤ 10%

只做辅助

# **⚖️ 7. 因子标准化（必须）**

# **方法**

z-score / tanh / min-max

# **统一范围**

\[-1, +1]

# **🧠 8. 因子融合（Composite Score）**

# **公式**

score = Σ (factor\_i × weight\_i)

# **推荐初始权重**

{

"trend": 0.3,

"flow": 0.25,

"sentiment": 0.2,

"macro": 0.15,

"behavioral": 0.1,

"historical": 0.05

}

# **🧠 9. 市场状态识别（Regime）**

# **输出**

RISK\_ON

RISK\_OFF

NEUTRAL

# **示例规则**

if trend > 0 && flow > 0:

RISK\_ON

if sentiment < -0.5 && macro < 0:

RISK\_OFF

# **🔗 10. 输出接口（对接决策系统）**

{

"composite\_score": 0.22,

"regime": "RISK\_OFF",

"factors": {...}

}

# **⚠️ 11. 关键设计约束**

# **❗ 1. 可回测**

所有因子必须可复现

（LLM输出必须缓存）

# **❗ 2. 低延迟**

因子计算 ≤ 秒级

# **❗ 3. 去噪**

使用EMA / rolling mean

# **🧪 12. 常见错误**

# **❌ 因子全用LLM**

→ 不可控

# **❌ 不做时间对齐**

→ 信号错乱

# **❌ 权重随意改**

→ 结果不稳定

# **🚀 13. 扩展方向**

链上数据（BTC交易所流入）

期权IV（隐含波动率）

美元指数

利率

# **🧠 最终总结（核心一句话）**

因子系统本质是：

用确定性模型建立“交易基础”

用 LLM 补充“信息理解能力”

# **🧠 一、为什么因子必须可扩展？**

# **1️⃣ 市场会变（这是本质）**

2021：牛市 → 动量有效

2022：熊市 → 宏观主导

2024+：ETF主导 → Flow最重要

👉 如果你写死：

trend + sentiment

👉 很快失效

# **2️⃣ 你一定会不断加新因子**

未来你几乎一定会加：

链上数据（BTC流入交易所）

期权IV

利率 / 美元指数

更多交易员数据

👉 如果不可扩展：

- 改一次代码 = 改全系统
- 权重全崩

# **3️⃣ 因子需要做实验（非常重要）**

你会做：

开/关某个因子

调整权重

AB测试

👉 没扩展性 → 做不了优化

# **🧩 二、正确的设计目标（工程级）**

你的因子系统必须满足：

✅ 插件化（plug-in）

✅ 可开关（enable/disable）

✅ 可配置权重

✅ 可独立测试

# **🏗️ 三、推荐架构（核心设计）**

# **👉 因子 = 标准模块（Plugin）**

每个因子都必须遵守统一接口：

输入：features

输出：score \[-1,1]

# **🔹 标准接口定义（核心）**

{

"name": "trend",

"enabled": true,

"weight": 0.3,

"compute": "function"

}

# **🧠 四、因子系统结构（强烈推荐）**

Factor Engine

├── Trend Factor

├── Flow Factor

├── Sentiment Factor

├── Macro Factor

├── Behavioral Factor

├── Historical Factor

👉 本质：

一个“因子管理器” + 多个“因子插件”

# **⚙️ 五、运行流程（标准）**

1. 加载所有因子（配置驱动）
2. 过滤 enabled = true
3. 每个因子独立计算
4. 标准化输出
5. 加权合成

# **🧩 六、配置驱动（非常关键）**

# **👉 因子不要写死在代码里**

用配置控制：

{

"factors": \[

{"name": "trend", "weight": 0.3, "enabled": true},

{"name": "flow", "weight": 0.25, "enabled": true},

{"name": "sentiment", "weight": 0.2, "enabled": true},

{"name": "macro", "weight": 0.15, "enabled": true},

{"name": "behavioral", "weight": 0.1, "enabled": false}

]

}

👉 好处：

不用改代码就能调策略

# **🧠 七、LLM因子怎么处理？**

# **👉 单独作为一个插件**

SentimentFactor（LLM）

但必须：

缓存输出（保证可回测）

结构化结果

可降级（LLM挂了也能跑）

# **🧠 八、向量因子怎么接？**

同样：

HistoricalFactor（向量 + LLM）

限制：

权重低（<10%）

可关闭

# **⚠️ 九、必须避免的错误设计**

# **❌ 写成一个大函数**

compute\_all\_factors()

👉 无法扩展

# **❌ 因子之间耦合**

trend 依赖 sentiment

👉 会崩

# **❌ 权重写死**

trend \* 0.3

👉 无法调参

# **🧠 十、进阶（专业级玩法）**

# **🔥 1️⃣ 动态权重（非常重要）**

牛市 → trend权重大

风险期 → macro权重大

# **🔥 2️⃣ 因子分组**

Trend Group

Flow Group

Macro Group

# **🔥 3️⃣ 因子评分监控**

你要能看到：

哪个因子赚钱

哪个因子在害你

# **🧠 十一、一句话总结**

👉 因子系统本质是一个“可插拔评分引擎”
