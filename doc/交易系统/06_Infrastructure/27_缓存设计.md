# 🧠 一、缓存定位

## 1.1 缓存作用

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│   数据层    │ ──→ │    缓存层    │ ──→ │   计算层    │
│  (PostgreSQL等)  │     │   (Redis)   │     │  (Service) │
└─────────────┘     └─────────────┘     └─────────────┘
                           │
                           ▼
                    ┌─────────────┐
                    │   前端      │
                    └─────────────┘
```

**缓存核心作用：**
- 减少数据库压力
- 提升响应速度
- 支撑实时计算

# 🎯 三、缓存策略

## 3.1 缓存数据分类

| 数据类型 | 缓存策略 | TTL | 说明 |
|----------|----------|-----|------|
| **实时行情** | Write-Through | 1s | 价格/深度 |
| **因子数据** | Write-Through | 60s | 因子分数 |
| **用户配置** | Read-Aside | 1h | 用户配置 |
| **会话数据** | Read-Aside | 24h | Session |
| **计算结果** | Write-Behind | 5min | 聚合数据 |
| **限流计数** | Write-Through | 1min | 计数器 |

## 3.2 缓存Key设计

```
# 命名规范：{service}:{entity}:{id}:{field}

# 行情数据
market:price:BTC:current        → BTC当前价格
market:depth:BTC:binance:5m    → 深度数据(5分钟)
market:kline:BTC:1h:current    → K线数据

# 因子数据
factor:score:BTC:current       → BTC因子分数
factor:regime:BTC:current      → BTC当前Regime
factor:composite:BTC:current    → BTC综合分数

# 用户数据
user:config:{user_id}          → 用户配置
user:session:{session_id}       → 会话数据
user:rate:{user_id}            → 限流计数

# 系统数据
system:health                  → 系统健康状态
system:feature:status          → Feature服务状态
system:factor:status           → Factor服务状态

# 缓存数据
cache:news:recent              → 最新新闻缓存
```

```python
# 价格缓存
async def cache_price(symbol: str, price: float):
    key = f"data:price:{symbol}"
    await redis.setex(key, 1, json.dumps({"price": price, "ts": time.time()}))

    key = f"data:price:{symbol}"
    data = await redis.get(key)
```

# 🔄 五、缓存一致性

## 5.1 更新策略

```python
# Write-Through（写入穿透）
async def update_price(symbol: str, price: float):
    # 1. 写数据库
    await db.update_price(symbol, price)
    # 2. 写缓存
    await redis.setex(f"data:price:{symbol}", 1, price)
    # 3. 发送Kafka
    await kafka.send("price_update", {"symbol": symbol, "price": price})

# Write-Behind（写入背后）
async def update_composite_score(symbol: str, score: float):
    # 1. 写缓存
    await redis.setex(f"factor:composite:{symbol}", 60, score)
    # 2. 异步写数据库
    asyncio.create_task(db.update_score(symbol, score))

# Read-Aside（旁路读取）
async def get_regime(symbol: str):
    # 1. 先查缓存
    cached = await redis.get(f"regime:current:{symbol}")
    if cached:
        return json.loads(cached)

    # 2. 缓存未命中，查数据库
    regime = await db.get_regime(symbol)

    # 3. 写缓存
    await redis.setex(f"regime:current:{symbol}", 60, json.dumps(regime))

    return regime
```

## 5.2 缓存失效策略

```python
# 主动失效（Event-Driven）
async def on_price_update(symbol: str):
    # 价格更新时，失效相关缓存
    await redis.delete(f"data:price:{symbol}")
    await redis.delete(f"feature:price:{symbol}")
    await redis.delete(f"factor:trend:{symbol}")

# 被动失效（TTL-Based）
# 所有缓存都有TTL，自动失效

# 批量失效（Pattern Match）
async def invalidate_pattern(pattern: str):
    keys = await redis.keys(pattern)
    if keys:
        await redis.delete(*keys)
```

## 5.3 缓存预热

```python
async def warmup_cache():
    # 启动时加载热点数据

    # 1. 加载实时行情
    symbols = ["BTC", "ETH", "SOL"]
    for symbol in symbols:
        price = await db.get_latest_price(symbol)
```

# 🛡️ 六、缓存安全

## 6.1 限流

```python
# 基于Redis的滑动窗口限流
async def rate_limit(user_id: str, limit: int = 100, window: int = 60) -> bool:
    key = f"user:rate:{user_id}"

    current = await redis.incr(key)

    if current == 1:
        await redis.expire(key, window)

    return current <= limit
```

## 6.2 熔断

```python
# 缓存熔断策略
class CacheCircuitBreaker:
    def __init__(self):
        self.failure_count = 0
        self.failure_threshold = 5
        self.circuit_open = False

    async def get(self, key: str):
        if self.circuit_open:
            # 熔断开启，直接查数据库
            return await db.get(key)

        try:
            value = await redis.get(key)
            self.failure_count = 0
            return value
        except Exception as e:
            self.failure_count += 1
            if self.failure_count >= self.failure_threshold:
                self.circuit_open = True
            return await db.get(key)

    async def set(self, key: str, value: str):
        if self.circuit_open:
            return await db.set(key, value)

        try:
            await redis.set(key, value)
            self.failure_count = 0
        except Exception as e:
            self.failure_count += 1
            if self.failure_count >= self.failure_threshold:
                self.circuit_open = True
            await db.set(key, value)
```
