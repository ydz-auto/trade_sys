## 三、核心架构设计

### 3.1 状态管理 (Zustand)

**设计原则：**
- 单一 Store 设计，集中管理所有交易相关状态
- 类型安全的状态更新
- 轻量级，避免 Redux 过度复杂

**核心 Store：** [tradingStore.ts](file:///Users/yangdezeng/00_crypto/00_trade_agent/20260506/frontend/src/store/tradingStore.ts)

```typescript
interface TradingState {
  // 系统模式
  mode: SystemMode;                              // 'BACKTEST' | 'SIMULATION' | 'LIVE'
  setMode: (mode: SystemMode) => void;
  
  // 市场数据
  prices: PriceData[];                           // 价格数据
  compositeScore: number;                        // 综合评分
  regime: RegimeState;                           // 市场状态
  risk: RiskIndex;                               // 风险指数
  signal: Signal;                                // 交易信号
  
  // 因子管理
  factors: Factor[];                             // 所有因子
  updateFactorWeight: (type: FactorType, weight: number) => void;
  
  // 仓位管理
  positions: Position[];                         // 当前持仓
  
  // 权重版本管理
  weightVersions: WeightVersion[];               // 历史权重版本
  currentVersion: string;                        // 当前版本
  setCurrentVersion: (version: string) => void;
  
  // 数据源状态
  dataSources: DataSourceStatus[];               // 各数据源状态
  
  // 交易员和社交数据
  traders: Trader[];                             # 交易员数据
  socialPosts: SocialPost[];                     # 社交帖子
  
  // 系统状态
  isConnected: boolean;                          # 是否连接后端
  lastUpdate: Date;                              # 最后更新时间
}
```

### 3.2 类型系统 (TypeScript)

完整的 TypeScript 类型定义见 [types/index.ts](file:///Users/yangdezeng/00_crypto/00_trade_agent/20260506/frontend/src/types/index.ts)

**核心类型：**

| `Position` | 持仓信息 |
| `WeightVersion` | 权重版本历史 |
| `SystemMode` | 系统模式 (BACKTEST, SIMULATION, LIVE) |

### 3.3 API 层设计

**API 模块：** [services/api/tradingApi.ts](file:///Users/yangdezeng/00_crypto/00_trade_agent/20260506/frontend/src/services/api/tradingApi.ts)

**设计亮点：**
- 支持真实 API 和 Mock 数据切换
- 统一的错误处理
- 类型安全的 API 响应

**环境变量：**
```bash
VITE_API_BASE_URL=/api/v1          # API 基础 URL
VITE_USE_MOCK_API=true             # 是否使用 Mock 数据
```

**核心 API 方法：**

```typescript
// 获取所有交易数据
fetchAllTradingData(): Promise<TradingData>

// 获取价格
fetchPrices(): Promise<PriceData[]>

// 获取因子
fetchFactors(): Promise<Factor[]>

// 获取市场状态
fetchRegime(): Promise<RegimeState>

// 获取风险指数
fetchRisk(): Promise<RiskIndex>

// 获取交易信号
fetchSignal(): Promise<Signal>

// 获取持仓
fetchPositions(): Promise<Position[]>

// 获取权重版本
fetchWeightVersions(): Promise<WeightVersion[]>

// 更新因子权重
updateFactorWeight(type: string, weight: number): Promise<void>
```

## 四、核心页面功能

### 1. DashboardPage (仪表盘)
- 综合评分展示
- 市场概览
- 快速信号
- 实时数据更新

### 2. WeightConfigPage (权重配置)
- 因子权重调整
- 权重版本管理
- A/B 测试配置
- 版本历史查看

### 3. RegimePage (市场状态)
- 当前市场 regime 展示
- Regime 历史图表
- Regime 切换预警

### 4. RiskPage (风险管理)
- 综合风险指数
- 风险因子分解
- 风险阈值设置
- 风险预警

### 5. DecisionPage (交易决策)
- 当前信号详情
- 决策理由展示
- 历史决策记录
- 决策回测分析

### 6. ControlPage (系统控制)
- 系统模式切换 (LIVE/SIMULATION/BACKTEST)
- 系统状态监控
- 紧急停止功能

### 7. PositionsPage (持仓管理)
- 当前持仓列表
- 盈亏统计
- 仓位调整
- 止盈止损设置

### 8. ExecutionPage (执行记录)
- 订单历史
- 执行详情
- 滑点分析
- 成交统计

## 九、设计亮点

1. **类型安全：** 完整的 TypeScript 类型系统，避免运行时错误
2. **状态管理简洁：** Zustand 轻量级，避免 Redux 的 boilerplate
3. **组件复用：** 统一的组件库和布局系统
4. **主题统一：** 深色主题专业感强，适合交易系统
5. **Mock 支持：** 可使用 Mock 数据快速开发，不依赖后端
6. **响应式设计：** 支持多种屏幕尺寸
