# 🔧 四、多源冗余策略

## 4.1 核心原则

```
主数据源 + 备用数据源 + 告警机制
         ↓
   主不可用时自动切换
```

# 📈 五、时效性等级

| 时效等级 | 数据类型 | 采集频率 | 延迟容忍 |
|----------|----------|----------|----------|
| 🔴 毫秒级 | 价格、订单簿 | WebSocket | 0 |
| 🟠 秒级 | K线、技术指标 | 实时 | 1-5s |
| 🟡 分钟级 | 新闻、社交媒体 | 1-5分钟 | 30s |
| 🟢 小时级 | ETF、宏观数据 | 每小时/每日 | 1小时 |
| 🔵 日级 | 财务数据、公告 | 每日 | T+1 |

# 🛡️ 六、可靠性保障

## 6.1 数据质量检查

```python
DATA_QUALITY_CHECKS = {
    "price": {
        "max_deviation_percent": 0.01,  # 单次价格波动不超过1%
        "stale_threshold_seconds": 5,     # 数据超过5秒视为过期
    },
    "news_cn": {
        "min_length": 30,               # 中文内容最少30字符
        "max_age_hours": 12,            # 中文新闻超过12小时不采
    },
    "news_en": {
        "min_length": 50,               # 英文内容最少50字符
        "max_age_hours": 24,            # 英文新闻超过24小时不采
    },
    "sentiment": {
        "confidence_threshold": 0.6,     # LLM置信度低于60%不使用
    },
    "crypto_stocks": {
        "max_deviation_percent": 0.05,   # 美股波动较大，允许5%
        "stale_threshold_seconds": 60,   # 股票数据60秒视为过期
        "trading_hours_only": true,      # 仅在美股交易时段采集
    }
}
```

## 6.2 告警规则

```python
ALERT_RULES = {
    "source_down": {
        "condition": "consecutive_failures >= 3",
        "action": "switch_to_fallback + notify"

    "data_stale": {
        "condition": "data_age > threshold",
        "action": "warn + use_stale_data_flag"

        "condition": "price_change > 5% in 1min",
        "action": "alert + verify + pause_if_unverified"
```
