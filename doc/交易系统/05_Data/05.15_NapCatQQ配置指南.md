# NapCatQQ 配置指南

## 快速开始

### 1. 下载 NapCatQQ

访问: https://github.com/NapNeko/NapCatQQ/releases

下载最新版本的 **NapCatQQ**，根据你的系统选择：
- Windows: `NapCat-Setup-*.exe`
- Mac: `NapCat-*.dmg`
- Linux: 查看文档

### 2. 安装并登录

1. 运行安装程序
2. 打开 NapCatQQ
3. 使用手机 QQ 扫码登录
4. 加入你想监控的 QQ 群

### 3. 开启 WebSocket 服务

在 NapCatQQ 中：

1. 点击设置 → 进阶设置
2. 找到 "WebSocket 服务"
3. 开启 "启用 WebSocket"
4. 确认端口是 `3001`（默认）
5. 保存并重启 NapCatQQ

### 4. 配置环境变量

复制 `.env.example` 为 `.env`：

```bash
cd backend
cp .env.example .env
```

编辑 `.env` 文件：

```ini
# 禁用模拟模式
QQ_USE_MOCK=false

# NapCatQQ WebSocket 地址
QQ_WS_URL=ws://127.0.0.1:3001

# 要监控的群号，多个用逗号分隔
QQ_WATCH_GROUPS=123456789,987654321
```

### 5. 运行测试

```bash
cd backend
python3 services/data_service/examples/qq_mvp.py
```

## 完整步骤

### 第一步：准备 QQ 账号

使用一个真实的 QQ 账号（推荐用小号）：
1. 注册或使用现有 QQ 账号
2. 登录手机 QQ
3. 加入要监控的群
4. 确保账号状态正常，不会被风控

### 第二步：安装 NapCatQQ

NapCatQQ 是基于 QQ NT 版本的轻量级框架，比 go-cqhttp 更稳定。

**Windows:**
```
下载 .exe 安装包直接运行
```

**Mac:**
```
下载 .dmg 镜像安装
```

**Linux:**
参考: https://napcat.nepsno.com/

### 第三步：配置 NapCatQQ

启动 NapCatQQ 后：

1. 扫码登录
2. 在设置中开启：
   - WebSocket 服务
   - 群消息监听
   - 好友消息监听（可选）

默认配置：
- WebSocket 地址: `ws://127.0.0.1:3001`
- HTTP API 地址: `http://127.0.0.1:3000`

### 第四步：获取群号

要监控某个群，需要知道群号：

1. 打开 QQ
2. 进入目标群
3. 点击群信息
4. 复制群号（纯数字）

### 第五步：测试连接

创建一个简单的测试脚本 `test_napcat.py`：

```python
import asyncio
import websockets
import json

async def test():
    uri = "ws://127.0.0.1:3001"

    try:
        async with websockets.connect(uri) as websocket:
            print("✅ 连接成功！")
            print("等待接收消息...")

            async for message in websocket:
                data = json.loads(message)
                print(f"\n收到消息: {json.dumps(data, indent=2, ensure_ascii=False)}")

    except Exception as e:
        print(f"❌ 连接失败: {e}")
        print("\n请检查:")
        print("  1. NapCatQQ 是否启动")
        print("  2. WebSocket 是否开启")
        print("  3. 端口是否正确 (默认 3001)")

asyncio.run(test())
```

运行：
```bash
python3 test_napcat.py
```

如果成功，在 QQ 群里发一条消息，你应该能看到输出。

### 第六步：运行我们的监控程序

```bash
cd backend

# 确保环境变量正确
export QQ_USE_MOCK=false
export QQ_WATCH_GROUPS=你的群号

# 运行 MVP
python3 services/data_service/examples/qq_mvp.py
```

## 常见问题

### Q: 连接失败怎么办？

A: 检查以下几点：
1. NapCatQQ 是否正在运行？
2. WebSocket 服务是否开启？
3. 端口是否是 3001？
4. 防火墙是否阻止？

### Q: 收不到群消息？

A: 确认：
1. QQ 号确实在群里
2. 群号配置正确
3. NapCatQQ 有群消息权限

### Q: 如何获取更多群消息？

A: 在 NapCatQQ 设置中：
- 开启 "群消息"
- 开启 "历史消息"（可选）

### Q: 可以同时监控多个群吗？

A: 可以！环境变量里用逗号分隔：
```
QQ_WATCH_GROUPS=123456,789012,345678
```

## 进阶：QQ → Telegram 转发

如果想把 QQ 群消息转发到 Telegram：

1. 创建 Telegram Bot: @BotFather
2. 获取 Bot Token
3. 获取 ChatID: @username_to_id_bot
4. 配置环境变量：

```ini
TELEGRAM_BOT_TOKEN=你的bot_token
TELEGRAM_CHAT_ID=你的chat_id
```

5. 运行：

```bash
python3 services/data_service/examples/qq_to_telegram.py
```

## NapCatQQ 配置文件参考

NapCatQQ 的配置通常在：
- Windows: `%APPDATA%/NapCat/config/`
- Mac: `~/Library/Application Support/NapCat/config/`

关键配置项（`config.yaml`）：
```yaml
WebSocket:
  enable: true
  host: 0.0.0.0
  port: 3001
```
