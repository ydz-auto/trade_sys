# Twitter Push Notification 架构

## 概述

利用 Chrome 浏览器通知作为 **P0 超快速通道**，实时监控 Twitter P0 账号的加密相关推文。

## 架构图

```
┌─────────────────────────────────────────────────────────────────────┐
│                        Twitter P0 监控架构                           │
└─────────────────────────────────────────────────────────────────────┘

                          P0 快速通道
                    ┌───────────────────────┐
                    │  Twitter 浏览器通知   │
                    │  (Twitter 主动推送)   │
                    └───────────┬───────────┘
                                │
                                ▼
                    ┌───────────────────────┐
                    │   Chrome Extension   │
                    │   (background.js)    │
                    └───────────┬───────────┘
                                │ WebSocket
                                ▼
                    ┌───────────────────────┐
                    │  WebSocket Server    │
                    │  (localhost:8765)    │
                    └───────────┬───────────┘
                                │
                                ▼
                    ┌───────────────────────┐
                    │     EventBus         │
                    │   StandardEvent      │
                    └───────────┬───────────┘
                                │
                                ▼
                    ┌───────────────────────┐
                    │   Kafka / LLM        │
                    │   事件处理            │
                    └───────────────────────┘
```

## 优势

| 特性 | 优势 |
|------|------|
| **实时性** | Twitter 主动推送，比轮询更快 |
| **低成本** | 免费，无需 API Key |
| **低风控** | 不依赖爬虫，不易被封 |
| **低延迟** | < 1 秒推送 |
| **高优先级** | Twitter 会优先推送 P0 账号 |

## 限制

| 限制 | 说明 |
|------|------|
| **只能监控账号** | 不能监控关键词 |
| **通知数量有限** | 关注太多账号会降频 |
| **P0 优先** | Twitter 会优先推送高互动账号 |

## 安装步骤

### 1. 启动 WebSocket 服务器

```bash
cd backend
PYTHONPATH=. python3 services/data_service/twitter_push_server.py
```

### 2. 安装 Chrome Extension

1. 打开 `chrome://extensions/`
2. 开启右上角 **Developer Mode**
3. 点击 **Load unpacked**
4. 选择 `frontend/extensions/twitter-push`

### 3. 登录 Twitter

1. 打开 Twitter/X
2. 登录你的账号
3. 确保开启浏览器通知

## 配置

### 修改 WebSocket 地址

编辑 `background.js` 中的配置：

```javascript
const CONFIG = {
    wsUrl: 'ws://localhost:8765/twitter-push',
    reconnectInterval: 3000,
    maxReconnectAttempts: 10
};
```

### 修改 P0 账号列表

编辑 `content.js` 中的 `P0_ACCOUNTS` 数组：

```javascript
const P0_ACCOUNTS = [
    'elonmusk',
    'cz_binance',
    // 添加更多账号
];
```

## 工作原理

### 1. Twitter 推送通知

当 P0 账号发推时，Twitter 会推送浏览器通知

### 2. Chrome Extension 监听

- `background.js`: 保持 WebSocket 连接
- `content.js`: 监控 Twitter 页面上的推文

### 3. WebSocket 转发

Extension 通过 WebSocket 将推文数据发送到服务器

### 4. 数据标准化

服务器将推文转换为 `StandardEvent` 格式

### 5. 事件分发

通过 EventBus 分发到策略/风控/通知引擎

## 熔断降级

```
主通道 (GraphQL) 挂了
         │
         ▼
┌─────────────────────┐
│  Push Notification  │
│  仍能接收 P0 推文   │
└─────────────────────┘
```

即使 GraphQL 监听器挂了，Push Notification 仍能保住 P0 通道。

## 数据格式

### WebSocket 消息

```json
{
    "type": "tweet",
    "source": "twitter_push",
    "timestamp": 1704936000000,
    "data": {
        "id": "1234567890",
        "author": "elonmusk",
        "content": "Bitcoin is the future! #BTC",
        "url": "https://twitter.com/elonmusk/status/1234567890",
        "likes": 125000,
        "retweets": 25000,
        "mentionedSymbols": ["BTC"]
    }
}
```

### StandardEvent

```python
StandardEvent(
    source="twitter",
    event_type="tweet",
    title="@elonmusk: Bitcoin is the future! #BTC...",
    content="Bitcoin is the future! #BTC",
    importance=0.85,
    sentiment="bullish",
    symbols=["BTC"],
    metadata={
        "author": "elonmusk",
        "tweet_id": "1234567890",
        "is_p0_account": True,
        "push_source": "chrome_extension"
    }
)
```
